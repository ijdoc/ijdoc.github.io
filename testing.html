Test Automation

Unlocking anxiety-free release cycles

I believe in fully automated test-based development (TBD). Scripted, reproducible and compliant test scripts and rigs are the best way to identify and fix issues quickly and effectively while keeping technical debt under control.

QA Testing Robot

When tecla started shipping its second generation flagship product, we were still testing every single shipped unit manually. Testing protocols were so long that we started skipping steps even though detailed scripts were available.

As part of the production scaling efforts, as CTO and Technical Lead, I assembled a robotic test rig that could complete all QA testing unattended, thus allowing a single shipping manager to complete QA testing for all units, as part of the shipping workflow.

Minesweeping React Native

As Technical Lead with Myplanet (2019 - present), I was dropped into a project with a leading Canadian Telco whose React Native app launch had been delayed by months due to stubborn bugs that no other developer had been able to address. Using Appium, I compiled integration tests to consistently trigger the offending behaviors, and used those tests to diagnose and resolve all outstanding issues in a few days.

Test Automation

Unlocking anxiety-free release cycles

Case Study: Minesweeping React Native

React Native is notorious for its hidden landmines: Inconsistencies and incompatibilities between node libraries and/or mobile device fragmentation can often obscure bugs driving developers mad.

As Technical Lead with Myplanet, I was dropped into a project with a leading Canadian Telco whose React Native app launch had been delayed by months due to stubborn bugs that no other developer had been able to address. Using Appium, I automated the behaviors into integration tests and used them to diagnose and resolve all outstanding issues in a few days.

Case Study: QA Testing with Robots

When tecla started shipping its second generation flagship product —I was CTO from 2010 to 2018—, we were still testing every single shipped unit manually. Testing protocols were so long that we started to accidentally skip steps even when detailed scripts were available.

As part of the production scaling efforts, I assembled a robotic test rig that could complete all QA testing unattended, allowing a single shipping manager to complete QA testing for all units, as part of the shipping workflow.
